# EderaON

EderaON is a free-to-use, single-node tier of [Edera](https://edera.dev),
designed to give security and platform teams hands-on experience with hardened
container runtime protection.

Edera isolates containers using the [Xen hypervisor](https://xenproject.org/)
instead of relying on shared kernel namespaces. Each container runs in its own
lightweight virtual machine (zone), providing hardware-level isolation that
prevents container escapes from reaching other workloads or the host.

EderaON gives you a one-node license valid for twelve months to install and
evaluate this technology on your own infrastructure.

## What you get

- **One-node license** valid for twelve (12) months
- **Edera runtime** with Xen-based container isolation
- **Access to Edera container images** via `images.edera.dev`
- **Community support** via [GitHub Issues](https://github.com/edera-dev/on/issues)

## AWS Quick Start

Provision an EC2 instance, install the Edera hypervisor, and run demos
comparing Docker's shared-kernel containers against Edera's VM-isolated zones.

### Prerequisites

- [AWS CLI](https://aws.amazon.com/cli/) configured with valid credentials
- An EderaON license key from [on.edera.dev](https://on.edera.dev)

### Setup

```sh
# 1. Set your Edera license key
export EDERA_LICENSE_KEY="<your-license-key>"

# 2. Provision infrastructure + install Edera (single command)
./scripts/aws/setup.sh
```

`setup.sh` validates prerequisites, creates all AWS resources (VPC, subnet,
security group, EC2 m5.large instance), installs Docker, installs the Edera
kernel, and runs a smoke test. Resource IDs are saved to
`scripts/aws/variables.sh` for later use.

To restore variables in a new shell:

```sh
source ./scripts/aws/variables.sh
```

### Connect to the Instance

```sh
./connect.sh
```

`connect.sh` is generated by the setup script with the correct key path and IP.

### Running Demos

Once connected to the instance:

```sh
# Run all demos
bash /home/ubuntu/demos/run-all.sh
```

See [demos/README.md](demos/README.md) for descriptions of each demo:

- **DEMO_00** — Verify Edera On installation (health check)
- **DEMO_01** — Container escape vs zone isolation (side-by-side comparison)
- **DEMO_02** — AI agent attack simulation (8-step automated escape attempts)

### Tear Down

```sh
# Deactivate your license first at https://on.edera.dev so you can reuse it
./scripts/aws/teardown.sh
```

### Manual Setup (Alternative Distros)

The setup script handles Ubuntu automatically. For Amazon Linux 2023:

```sh
sudo dnf install -y docker nftables
sudo systemctl start docker
sudo usermod -aG docker ec2-user
```
